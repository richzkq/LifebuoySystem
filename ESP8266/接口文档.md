# 智能救生圈 — ESP8266 舵机控制接口文档

## 1. 连接信息

| 项目 | 值 |
|------|-----|
| 协议 | MQTT 3.1.1 |
| 服务器 IP | `47.83.199.93` |
| 端口 | `1883` |
| 认证 | 无需用户名密码（匿名连接） |
| Client ID | 自定义，建议 `lifebuoy-esp8266` + MAC 地址后4位 |

## 2. 订阅 Topic

```
lifebuoy/servo/+/command
```

`+` 是 MQTT 单层通配符，匹配任意 deviceId（如 `rocket001`）。

也可以订阅指定设备：

```
lifebuoy/servo/rocket001/command
```

## 3. 消息格式

**JSON，QoS 1（至少一次送达）**

```json
{
  "cmd":       "RELEASE",
  "deviceId":  "rocket001",
  "reason":    "CONSECUTIVE_DROWNING",
  "ts":        1717320000000
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `cmd` | String | 是 | 命令类型。当前只有 `RELEASE`（释放救生圈） |
| `deviceId` | String | 是 | 触发报警的设备 ID |
| `reason` | String | 是 | 触发原因：`CONSECUTIVE_DROWNING`（连续帧溺水检测）、`MANUAL_TRIGGER`（手动触发） |
| `ts` | Long | 是 | 触发时间戳（毫秒） |

## 4. 舵机行为

| 步骤 | 动作 | 时机 |
|------|------|------|
| 收到 RELEASE | 舵机转到 90°（释放位） | 立即 |
| 保持释放 | 舵机停在 90° | 2 秒 |
| 自动复位 | 舵机转回 0°（锁定位） | 释放后 2 秒 |
| 防重复 | 已释放状态下忽略新指令 | — |

## 5. 触发方式

### 5.1 自动触发（连续溺水检测）

服务器检测到连续 5 帧溺水 → 自动发布 RELEASE 消息。

### 5.2 手动触发（测试用）

```bash
curl -X POST "http://47.83.199.93:8080/api/servo/trigger?deviceId=rocket001"
```

### 5.3 查看状态

```bash
curl "http://47.83.199.93:8080/api/servo/status"
```

返回示例：
```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "rocket001": {
      "deviceId": "rocket001",
      "consecutiveCount": 0,
      "servoReleased": true,
      "lastTriggerTime": 1717320000000,
      "threshold": 5,
      "cooldownMs": 30000
    }
  }
}
```

## 6. 验证方法

在服务器上监听 MQTT 消息：

```bash
# 安装工具
sudo apt install mosquitto-clients -y

# 监听舵机 topic
mosquitto_sub -h localhost -p 1883 -t "lifebuoy/servo/+/command" -v

# 另开终端，手动触发
curl -X POST "http://localhost:8080/api/servo/trigger?deviceId=rocket001"
```

当你手动触发后，`mosquitto_sub` 会输出：

```
lifebuoy/servo/rocket001/command {"cmd":"RELEASE","deviceId":"rocket001","reason":"MANUAL_TRIGGER","ts":1717321123456}
```

这就说明舵机推送接口正常工作。
